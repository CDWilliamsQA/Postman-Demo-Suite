# Contributing Guidelines

Thank you for your interest in contributing to this project.

## How to Contribute

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test locally using Newman
5. Submit a pull request

---

## Development Workflow

This project uses Postman collections and Newman CLI for automation.

Before submitting changes:

- Ensure tests pass locally
- Maintain existing test structure
- Follow current naming conventions
- Keep environment variables configurable

---

## Code Style

- Keep test scripts readable and structured
- Use descriptive test names
- Avoid hardcoded values where possible
- Follow existing folder structure

---

## Pull Requests

PRs should include:

- Clear description of changes
- Reason for change
- Screenshots or logs if applicable
- Confirmation tests pass

---

## Reporting Issues

Use GitHub Issues to report:

- Test failures
- Documentation errors
- Suggestions for improvement
